# Stage 1: Pokémon TCG Sets Reference Dataset

## Status

Completed.

This stage created the first clean reference dataset for the project: `pokemon_tcg_sets`.

The purpose of this stage was to build a reliable local reference table for official Pokémon TCG sets using the Pokémon TCG `en.json` source file.

This stage intentionally does **not** include Cardmarket mapping, Cardmarket expansion IDs, German names, slugs, or mapping confidence.

---

## Stage Goal

Create a clean, validated PostgreSQL / Supabase reference table for official Pokémon TCG sets.

The workflow is:

```text
Pokémon TCG en.json
        ↓
Python transformation script
        ↓
pokemon_tcg_sets.csv
        ↓
PostgreSQL / Supabase table
        ↓
SQL validation checks
```

---

## Main Decision

`pokemon_tcg_sets` is a local project reference table, not a raw copy of the source JSON.

The source JSON is used to populate and validate the dataset, but the project owns the final table structure.

Cardmarket-specific fields are intentionally excluded from v1.

Excluded from Stage 1:

- Cardmarket `idExpansion`
- German set names
- Cardmarket slugs
- Cardmarket URLs
- Cardmarket mapping confidence
- Cardmarket expansion reference logic

These may be added later through a separate mapping stage or migration.

## Transformation Logic

The Python loader maps source JSON fields to local CSV fields:

| Source JSON Field | Local Field |
|---|---|
| `id` | `pokemon_tcg_set_id` |
| `name` | `name_en` |
| `series` | `series_en` |
| `ptcgoCode` | `ptcgo_code` |
| `releaseDate` | `release_date` |
| `printedTotal` | `printed_total` |
| `total` | `total_cards` |
| `legalities` | `legalities` |
| `images.symbol` | `symbol_url` |
| `images.logo` | `logo_url` |
| `updatedAt` | `source_updated_at` |

Additional local fields:

| Field | Logic |
|---|---|
| `secret_card_count` | `max(total_cards - printed_total, 0)` |
| `is_active` | `true` |
| `notes` | Empty unless a source anomaly is detected. |

---

## Important Data Quality Decision

The original idea was:

```text
secret_card_count = total_cards - printed_total
```

During import, this produced negative values for some promo sets.

Final Stage 1 logic:

```text
secret_card_count = max(total_cards - printed_total, 0)
```

If the source has:

```text
total_cards < printed_total
```

then:

```text
secret_card_count = 0
```

and the reason is stored in `notes`.

This keeps the analytical field safe while preserving the source anomaly.

---

## Schema Decision

The schema keeps hard constraints for safe universal rules:

- primary key must exist
- names must exist
- numeric card counts must be non-negative
- `secret_card_count` must be non-negative

The schema does **not** enforce:

```text
total_cards >= printed_total
```

Reason:

Some promo or special sets in the upstream Pokémon TCG source have `total_cards` lower than `printed_total`.

This is handled as a diagnostic check, not as a blocking database constraint.

---

## Import Process

The CSV was imported into Supabase manually through the Table Editor.

Final workflow:

```text
1. Run schema file.
2. Generate CSV with Python loader.
3. Import CSV into `public.pokemon_tcg_sets`.
4. Run validation checks.
5. Document results.
```

---

## Validation Result

The import completed successfully.

Validation confirmed:

- no missing set IDs
- no missing English names
- no duplicate primary keys
- no negative card count values
- no missing `created_at` values
- no missing `updated_at` values

Known source anomalies detected:

| pokemon_tcg_set_id | name_en | printed_total | total_cards | secret_card_count |
|---|---|---:|---:|---:|
| `swshp` | SWSH Black Star Promos | 307 | 304 | 0 |
| `svp` | Scarlet & Violet Black Star Promos | 102 | 75 | 0 |

These rows are allowed.

They are preserved because they are valid upstream source records. The anomaly is documented in the `notes` field.

## What Stage 1 Does Not Do

Stage 1 does not:

- download data automatically
- map Pokémon TCG sets to Cardmarket expansions
- create Cardmarket expansion references
- create German names
- create Cardmarket slugs
- create Cardmarket URLs
- load price guide data
- build BI views
- track personal collection items
- generate analytics signals

This is intentional.

The goal of Stage 1 was only to create a stable Pokémon TCG sets reference foundation.
